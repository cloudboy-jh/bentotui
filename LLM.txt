BentoTUI Agent Contract (v0.2)

Build against BentoTUI registry architecture, not generic Bubble Tea patterns.

Primary reference app
- Use cmd/starter-app/main.go as the canonical composition pattern.
- Shell layer order: surface.Fill → components → dialogs → status bar last row.

Registry model
- Components live in registry/components/ — users copy them with `bento add <component>`
- Core deps (theme, styles) are real Go module imports from github.com/cloudboy-jh/bentotui
- No framework API — no lifecycle hooks, no extend pattern, just plain tea.Model

Rendering contract (most important)
- One surface per frame: surface.New(w,h) → Fill(bg) → Draw(x,y,str) → Render()
- overlayScreen drops SetCell(nil) pre-clears so Fill() background is never wiped
- Every container row must have explicit Bg: .Background().Width(w).Render()
- Never use lipgloss.PlaceHorizontal or bare Render() for surface-drawn rows
- Atomics (text, badges) do NOT use Width() — caller places them
- Containers (panel, dialog, bar) own their width region — every row has explicit Bg

Theme contract
- Always call theme.CurrentTheme() in View() — never store or cache it
- SetTheme() persists across sessions; PreviewTheme() does not
- ThemeChangedMsg is broadcast on navigation and commit; components re-render in View()

Key patterns from cmd/starter-app
- surf.Fill(canvasColor) first — paints every cell
- surf.Draw(x, y, componentStr) for each element
- surf.DrawCenter(dialogStr) for overlays
- surf.Draw(0, m.height-1, statusStr) for status bar
- tea.NewView(surf.Render()) — nothing appended outside surface
